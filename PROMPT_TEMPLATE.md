# PROJECT HADES 提示词设计框架

> **这是本项目的核心资产。**  
> 它是一份「用自然语言精确描述视觉设计 + 交互逻辑」的完整提示词模板。  
> 任何想做**暗黑科幻 / 电影级叙事网页**的人，都可以参照这个框架来写自己的提示词。

---

## 这个框架能干什么？

把你对一个网页的**所有想象**——视觉风格、动画细节、交互逻辑、技术参数——用自然语言完整描述出来，让 AI 直接生成可运行的代码。

**核心思想**：不讲「用什么技术」，而是讲「最终效果是什么」。AI 会自动选技术实现。

---

## 框架结构（共 7 个模块）

```
模块 1：媒体素材提示词（视频 + 图片）
   ↓
模块 2：全局设置（HTML 结构 + 基础 CSS）
   ↓
模块 3：第一页（主视觉 landing）
   ↓
模块 4：第二页（内容/探索页）
   ↓
模块 5：滚动过渡 JS（page transition）
   ↓
模块 6：其他 JS（粒子、交互、控制）
   ↓
模块 7：文件编码与交付要求
```

## 模块 1：媒体素材提示词

> **用途**：让 AI（或设计师）先生成项目所需的图片和视频，再把路径替换进代码。

### 写法要点

| 要点 | 说明 | 示例 |
|------|------|------|
| **分时段描述** | 视频按秒分段写，每段写清楚画面内容 | `0-2秒（铺垫）... 2-6秒（高潮）...` |
| **视觉参数** | 写清光影、质感、颜色、粒子效果 | `淡蓝光晕`、`炭黑色干枯粗糙的金属与岩石混合肌理` |
| **氛围词** | 用感性语言锁定「感觉」 | `深空死寂氛围感拉满`、`动静落差收尾` |
| **技术规格** | 写清比例、格式、风格禁忌 | `16:9 横向宽图`、`无卡通、无3D动画、无塑料光滑质感` |

### 模板（可直接复用）

```
### [素材名称] 提示词

[时间段/整体描述]
[画面内容详细描述，包括：主体、动作、光影、粒子效果、氛围]

技术规格：
- 比例：[16:9 / 9:16 / 其他]
- 风格：[写实 / 科幻 / 暗黑 / 其他]
- 禁忌：[无卡通、无文字、无XX]
- 保存路径：assets/[文件名]
```

## 模块 2：全局设置

> **用途**：定义 HTML 结构、引入的外部资源、全局 CSS 变量。

### 必须包含的内容

```
1. DOCTYPE + html lang（语言标签）
2. meta charset + viewport（编码 + 响应式）
3. Google Fonts 引用（把所有用到的字体列出来）
4. 自定义字体（@import 方式）
5. 全局 CSS Reset（* { margin:0; padding:0; box-sizing:border-box }）
6. html 根设置（scroll-behavior、scroll-snap-type）
7. body 基础样式（background、overflow、color、font-family）
8. :root CSS 变量（把所有会变的数值抽成变量）
9. .page 基础类（每页的公用样式）
```

### 写法要点

- **字体**：把 Google Fonts 的 URL 完整贴出来，包含所有 `wght@` 轴
- **CSS 变量**：给变量起有意义的名字，比如 `--p1fade`、`--mx`、`--my`
- **scroll-snap**：`scroll-snap-type:y proximity` 实现垂直分页滚动

## 模块 3：第一页（主视觉 Landing）

> **用途**：定义网站的第一屏，通常是「品牌展示 + 核心视觉冲击」。

### 页面结构模板

```
【背景层】（从下到上，z-index 递增）
  ├── 视频/图片背景（z-index:0）
  ├── 全局渐变遮罩（z-index:1）
  ├── 顶部渐变（z-index:3）
  ├── 底部渐变（z-index:3）
  ├── 额外径向渐变（z-index:4，可选）
  └── UI 内容层（z-index:5~6）

【UI 内容层】
  ├── 左上 Logo + 标语
  ├── 右上导航
  ├── 中央主标题（通常是最醒目的元素）
  ├── 左侧标题块（副标题 + 正文）
  ├── 左侧按钮
  ├── 右下活动信息
  └── 右侧竖排小字（装饰性）
```

### 每个 UI 元素的描述格式

```css
/* 元素名称【位置】 */
选择器{ /* 布局属性 */ }
/* 子元素 / 伪元素 */
选择器::before/::after{ /* 内容 */ }
/* 动画 */
@keyframes 动画名{ 0%{...} 100%{...} }
```

### 关键设计模式

| 模式 | 作用 | 实现方式 |
|------|------|----------|
| **魂环** | 元素周围的发光光环，增加「灵性」 | `::before` + `radial-gradient` + `ringExpand` 动画 |
| **流光** | 文字表面的光泽扫过效果 | `::after` + `linear-gradient` + `background-position` 动画 + `mix-blend-mode:screen` |
| **入场动画** | 页面加载时元素依次出现 | `opacity:0` → `forwards` + 不同 `animation-delay` |
| **无限脉冲** | 元素永恒呼吸的光效 | 独立的 `@keyframes` + `infinite` |

## 模块 4：第二页（内容/探索页）

> **用途**：定义网站的「内容页」，通常是交互式探索、信息展示。

### 本项目的核心交互

**鼠标跟随遮罩**：
- 第二页有一张「生晶」图，通过 `mask-image:radial-gradient` 只在鼠标位置显示
- JS 监听 `mousemove`，实时更新 CSS 变量 `--mx`、`--my`

```js
// 伪代码
page2.addEventListener('mousemove', (e) => {
  let rect = page2.getBoundingClientRect();
  mask.style.setProperty('--mx', (e.clientX - rect.left) + 'px');
  mask.style.setProperty('--my', (e.clientY - rect.top) + 'px');
});
```

### 页面结构模板

```
【背景层】
  ├── 基底图片（z-index:0）
  ├── 星尘粒子（z-index:1，JS 生成）
  ├── 鼠标跟随遮罩（z-index:2）
  ├── 顶部渐变（z-index:2）
  └── UI 层（z-index:3）

【UI 内容层】
  ├── 左侧竖排小字（装饰）
  ├── 主标题（逐字生成动画）
  ├── 副标题
  ├── 右下三列小字（数据/状态展示）
  └── 按钮（点击跳转其他页）
```

## 模块 5：滚动过渡 JS

> **用途**：实现两页之间的平滑过渡，用 `scroll-snap` + CSS 变量控制淡入淡出。

### 原理

```
用户滚动
  ↓
计算滚动位置 scrollY / viewport Height = 进度 p (0~1)
  ↓
根据 p 计算各页的 opacity（用 CSS 变量传递）
  ↓
CSS 用 var(--p1fade) / var(--p2fade) 控制显示/隐藏
```

### 关键代码模板

```js
window.addEventListener('scroll', () => {
  let sy = window.scrollY;
  let vh = window.innerHeight;
  let p = Math.max(0, Math.min(1, sy / vh));
  
  let p1fade = 1 - Math.max(0, p - 0.05) / 0.5;
  let p2fade = Math.max(0, Math.min(1, (p - 0.45) / 0.4));
  
  document.documentElement.style.setProperty('--p1fade', p1fade.toFixed(3));
  document.documentElement.style.setProperty('--p2fade', p2fade.toFixed(3));
}, { passive: true });
```

## 模块 6：其他 JS

> **用途**：定义粒子系统、视频控制、自定义缓动等辅助功能。

### 常用功能清单

| 功能 | 实现方式 |
|------|----------|
| **星尘粒子** | JS 循环创建 40 个 `div`，随机大小/位置/时长/延迟 |
| **逐字生成** | 遍历字符串，每个字符包一个 `span`，分别设 `animation-delay` |
| **视频可见性** | `IntersectionObserver`，进入视口 `play()`，离开 `pause()` |
| **自定义缓动** | `cubic-bezier(.2,1,.3,1)` 或自定义 `easeInOutCubic` 函数 |
| **语言切换** | `data-i18n` 属性 + JS 词典对象，切换时遍历 DOM 替换 |

## 模块 7：文件编码与交付

> **用途**：确保生成的文件在你的环境中能正常运行。

### 关键点

```
1. 必须 UTF-8 无 BOM（否则中文会乱码）
2. 写完验证前 3 字节不是 239,187,191（BOM 特征）
3. 单文件架构（所有 CSS/JS 内联进 HTML）
4. 双击浏览器即可打开运行（零依赖）
5. 媒体文件路径用相对路径（方便移植）
```

## 如何复用这个框架？

### 做新项目时，按这个顺序改：

```
1. 替换【媒体素材提示词】
   → 按你的新主题，重写视频/图片提示词

2. 替换【全局设置】的字体和颜色
   → 改 Google Fonts URL
   → 改 :root 里的主色调（现在用深空蓝 rgba(100,180,255)）

3. 重写【第一页】的所有文案和视觉元素
   → 保留动画结构，换内容

4. 重写【第二页】的交互逻辑
   → 可以保留鼠标跟随，也可以换成别的交互

5. 调整【滚动过渡】的分页点
   → 如果页数变了，改 JS 里的 p1fade/p2fade 计算公式

6. 检查【文件编码】
   → 确保输出的 HTML 是 UTF-8 无 BOM
```

## 这个框架的设计哲学

> **「感觉优先，技术其次」**

好的提示词不是写「用 CSS animation 实现…」，而是写「这个元素应该给人一种什么感觉」。

- ❌ 坏示例：`用 @keyframes 实现一个 3 秒的淡入动画`
- ✅ 好示例：`元素在页面加载后 0.3 秒开始淡入，持续 0.8 秒，使用 ease-out 缓动，给人一种「从虚空缓缓浮现」的感觉`

AI 会自动把「感觉」翻译成正确的技术实现。

如果你也想用这种方式做网页：
1. 先想清楚「感觉」和「氛围」
